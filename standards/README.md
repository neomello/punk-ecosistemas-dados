# Standards

Personal development standards for this workspace.

## Files

- `markdown.rules.md` — Markdown formatting rules
- `ai.rules.md` — AI assistant behavior rules
- `zshrc.rules.md` — Shell configuration patterns
- `mermaid.rules.md` — Mermaid diagram patterns
- `readme.template.md` — README template
- `readme.signature.md` — Author signature

## Critical Rules

1. **ALWAYS** add blank line after Markdown headers
2. **ALWAYS** use 2 spaces for indentation
3. **NEVER** commit secrets or API keys
